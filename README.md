# Frog Perch: Neural Network Pipeline

This repository contains the neural network training pipeline for the Frog Perch acoustic monitoring system. It acts as a feature-extraction and probabilistic inference engine. The outputs are designed to be calibrated downstream and fed into a process-based Bayesian hierarchical model for ecological call intensity estimation.

## Model Architecture: Windows vs. Slices

This network acts as a downstream temporal model on top of pre-trained Perch (v2) spatial embeddings. 

For every 5-second audio window, the Perch model outputs embeddings that represent 16 distinct temporal slices. Our downstream model relies on a single learned temporal representation that outputs probabilities at this slice-level. All window-level outputs are mathematically derived from these base slices:

* slice_probs (Slice-Level): The learned base Bernoulli probabilities for each of the 16 time slices within the 5-second window.
* count_probs (Window-Level): A Poisson-binomial probability distribution (0 to 16+ calls) deterministically aggregated from the 16 slice_probs via the SoftCountFromSlices layer.
* binary (Window-Level): The probability that at least one call exists anywhere in the 5-second window, derived as 1.0 - P(count=0).

## Training Dynamics & Structural Regularization

The network is trained using target labels that are already soft distributions, generated by overlapping acoustic annotations and confidence-weighted by bandwidth/duration statistics.

The primary loss function is the KL Divergence on the window-level count_probs distribution. 

The window-level binary and slice-level targets are used strictly for structural regularization. A LossAnnealingCallback linearly decays the weights of the binary and slice cross-entropy losses to 0.0 over the course of training, leaving the network to optimize solely for the KL divergence of the expected call count.

## Configuration

We enforce a strict separation between reusable source code and experimental configuration. 

* Static Config (configs/nn_config.py): Controls network dimensions, regularization, paths, and training variables.
* Dynamic Stats (configs/normalization.yaml): Contains auto-generated statistics (mean/std for confidence weighting). These are dynamically loaded at runtime.

## Workflow (Incomplete)

1. Data Normalization: Run the dataset normalization preprocessing script to compute the mean and standard deviation of call bandwidths and durations. This outputs configs/normalization.yaml.
2. Configure: Open configs/nn_config.py to define your paths, network size, and training dynamics.
3. Train: Run `python main.py` from the root directory. The dataset builder streams data infinitely and groups files by their embedded timestamp to prevent data leakage across partitions.
4. Evaluate: The pipeline saves the best model weights to checkpoints/best.keras. Performance is evaluated using Count KL Divergence, Expected Count MAE, and AUC.
